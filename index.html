What is GraphQL

API specification open sourced by Facebook in 2015

API connection layer specification developed and used at Facebook since 2012 and open sourced in 2015, with reference implementation in JavaScript

Not opinionated about network layer, payload format, or architecture, on the surface simply a query language

GraphQL schema

Between the front and back end sits the schema which defines the strongly typed interface of the API. Describes what types and shape of data can be requested by the client. Single source of truth.

Request

Practically speaking, front end app makes a single request to a graphql endpoint POSTing a graphql query string and receive a JSON object in exact shape we requested.

Benefits of GraphQL

Front-end driven

Declarative data fetching.

At the core of it is the query language, Client decides what to request, specific queries and responses in same shape (query language)

Client shapes the query. Needs of the front end drives queries, backend doesn't need to explicitly cater for front end requirements changing.

No overfetching

less traffic, no useless data

No underfetching

no implicit assumptions on data structures. Get what you request, or request fails.

front end handles optional data explicitly, which is specified by the schema

Nested data structures

querying for relationships

Oftentimes a client app needs to request several resources. In REST client might have to request multiple endpoints to be able to tell what to request from a different endpoint etc, unnecessary waterfall network traffic, as we wait for things to come back to be able to to tell what to request again.

Unopinionated

It’s something that can be built on top of your existing REST APIs and provided alongside your current REST endpoints

Developer workflow benefits

Query “composability"

Introspection

Automatically documented

Strongly typed, can be validated at compile time

Tooling, like GraphiQL, thanks to strongly typed and introspective schema, again enabling developers to validate queries and expore the API

  1. We're able to compose elements of queries anywhere in the query tree
  2. Able to inspect the schema to learn about the data and content types

Technology agnostic

Facebook uses it with React and it’s mostly used as such but it’s agnostic of this and can be used with whatever client side implementation.

It’s a programming language agnostic interface

Note: I know there's a lot of focus on React these days in Drupal community as it's being used in the UI, however GraphQL is not
